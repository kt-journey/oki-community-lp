# デザインシステム: Okey-Dokey LP

## 1. デザインコンセプト
- テーマ: Dark & Cinematic（夜明け前の隠岐の海）
- トーン:
  - 静寂
  - 熱量
  - 誠実
  - 企ての余白
- 運用方針:
  - デザインシステムは暫定版として運用し、正式ローンチ時に部分更新を前提とする
  - レイアウト基盤とコンポーネント構造は維持し、コピーと一部トーンのみ可変にする

## 2. カラーパレット
- Primary Background: `#0A131A`（Midnight Oki）
- Accent / Action: `#FF7A00`（Ember Orange）
- Sub Text: `#E0E0E0`（Foggy White）

### 推奨配色ルール
- 背景はPrimaryを基調に、セクション内で濃淡差をつける
- CTA・重要数字（50%）・状態ラベルの強調にAccentを使用
- 本文はSub Textを基本にし、階層に応じて透明度で情報量を調整

## 3. タイポグラフィ
- Headings: しっぽり明朝（Serif）
- Body: Noto Sans JP（Sans-serif）
- Monospace: Space Mono

### 文字運用ルール
- H1/H2はSerifで情緒と物語性を演出
- 本文・説明文はNoto Sans JPで可読性を優先
- STATUS表示やシステム的文言はSpace Monoで未完成感を演出

## 4. コンポーネント指針
- UIライブラリ: Tailwind CSS + shadcn/ui
- 主要コンポーネント:
  - CTAボタン
  - データ強調テキスト（50%）
  - 活動カード（4種）
  - ステータスラベル（PENDING...）
  - フッターリンク群（法的ページ）

### フェーズ移行を見据えた設計ルール
- CTAボタンは文言・リンク先・イベント名を設定値で差し替え可能にする
- セクション見出しと本文はコンテンツ定義ファイルから供給できる構造にする
- 強調表現（例: 50%）は将来別数値に差し替え可能な実装にする

## 5. エフェクト/モーション
- フィルムグレイン（軽量なオーバーレイ）
- Typed.js風テキスト演出
- Intersection Observerによるスクロール出現アニメーション
- ボタンホバー時のノイズ/グリッチ
- 動画利用時は自動再生

## 6. レイアウト方針
- モバイルファースト
- 基本構成: 1カラム中心（スマホ）→ 必要に応じてPCで多カラム化
- セクション間には余白を多めに確保し、読後の呼吸感を作る
- プレローンチと正式ローンチで共通レイアウトを使い、改修コストを抑える

## 7. アセット運用
- ヒーロー素材は初期段階でUnsplash等の仮素材を利用
- 本素材へ差し替えやすいよう、URLとコンポーネント責務を分離する

## 8. アクセシビリティ方針
- コントラストを担保（背景と本文）
- フォーカスリングの明示
- `prefers-reduced-motion` 対応は任意（必須ではない）
