# 要件定義書: Okey-Dokey LP

## 1. プロジェクト概要
- プロジェクト名: Okey-Dokey（隠岐移住者コミュニティ構想）
- 目的: 隠岐の島町における移住者の孤独を減らし、定着率50%の壁を突破する信頼インフラを構築する
- LPのゴール（プレローンチ）: 公式LINEへのウェイトリスト登録
- LPのゴール（正式ローンチ後）: サービス申込の獲得
- ターゲット: 隠岐の島町に住む既存移住者（ヨコのつながりを求める層）
- 戦略:
  - 運営元の秘匿（シークレット感）
  - 未完成な共創ストーリー
  - データ提示による危機感共有

## 2. フェーズ戦略
- Phase A（プレローンチ）:
  - 目的: 関心層の獲得と待機ユーザーの蓄積
  - 主CTA: LINEウェイトリスト登録
  - コピー方針: 問いかけ・危機感共有・共創の余白
- Phase B（正式ローンチ以降）:
  - 目的: 申込獲得とオンボーディング導線への接続
  - 主CTA: サービス申込
  - コピー方針: 提供価値・具体機能・参加メリットの明確化
- 共通方針:
  - レイアウトとコンポーネントを流用し、文言とCTAの差し替えで移行できる実装にする

## 3. ゴールとKPI
- 主要KPI（Phase A）: 公式LINE登録数
- 主要KPI（Phase B）: サービス申込数
- 副次KPI:
  - CTAクリック率（Hero / Footer）
  - セクション到達率（Section 1, 3, 5）
  - ページ滞在時間

## 4. ページ構成（情報設計）
- Section 0: Hero
  - コピー: 「隠岐の暮らし、このままでいいんだっけ？」
  - サブコピー: 移住者が居場所を自分たちで作る秘密プロジェクトの始動を訴求
  - CTA: 「プロジェクトの裏側を覗く」
- Section 1: The Silent Crisis
  - 独自調査に基づく危機提示
  - 「50%」を強調表示
- Section 2: Concept
  - 都会の流儀（お金）と隠岐の流儀（信頼）の対比
- Section 3: Activities
  - 魚突き・漁船体験
  - BBQ・キャンプ
  - 送迎相談シャトル
  - 身元保証
- Section 4: Co-creation
  - 未決定要素（名前、会費、ルール）を共創の余白として提示
  - STATUS: PENDING... 表現
- Section 5: Footer CTA
  - 「最初の100人」訴求
  - CTA: 「公式LINEで会議に参加する」

## 5. CTA要件
- HeroとFooterのCTAは同一URLを使用する
- 遷移先URL: `https://lin.ee/TJIOxNu`
- 正式ローンチ時はCTA遷移先と文言を申込導線へ置換する
- 置換容易性のため、CTAのURLと文言は設定値で管理する

## 6. コンテンツ要件
- データの出典表記: 「独自調査」を明示
- 言語: 日本語のみ
- ヒーロー素材:
  - 初期はUnsplash等の仮素材を利用
  - 将来的に差し替え可能な構造にする
- 正式ローンチ時は以下を差し替え対象とする:
  - Heroメインコピー
  - 問題提起中心の文脈
  - Footer CTA文言

## 7. 法務・運用要件
- LPフッターに別ページリンクを設置
  - プライバシーポリシー
  - 特定商取引法表記
  - 運営者情報
- 初期状態はひな形文面で公開可能にする

## 8. 非機能要件
- レスポンシブ: モバイルファースト
- 公開前提: HTTPS
- パフォーマンス:
  - 画像最適化（圧縮・適切サイズ）
  - 必要に応じて動画遅延ロード
- アクセシビリティ:
  - コントラスト担保
  - キーボードフォーカス可視化
  - `prefers-reduced-motion` 対応は必須要件としない

## 9. 計測要件
- 導入対象:
  - GA4
  - GTM
  - Meta Pixel
- 最低計測イベント:
  - CTAクリック（Hero / Footer）
  - LINE遷移クリック
- 正式ローンチ後の追加イベント（予定）:
  - 申込開始クリック
  - 申込フォーム到達
  - 申込完了
